1)Задание 11. Жадный алгоритм: раскраска графа
Условие. Раскрасить вершины графа в минимальное число цветов так, чтобы соседние вершины имели разные цвета (жадный подход).
Алгоритм: последовательная раскраска с выбором минимального доступного цвета.
Язык примера: Python
def greedy_coloring(adj_list):
 n = len(adj_list)
 colors = [-1] * n # -1 означает, что цвет не назначен
 for v in range(n):
#ДОПИСАТЬ: собрать используемые цвета соседей
 used = set()
 for neighbor in adj_list[v]:
 if colors[neighbor] != -1:
 used.add(colors[neighbor])
#ДОПИСАТЬ: назначить минимальный доступный цвет
 return colors
Что дописать: цикл поиска минимального натурального числа, не входящего в used.

2)Описание алгоритма:
 1. Собираем цвета уже раскрашенных соседей: проходим по списку смежных вершин adj_list[v] и, если сосед уже имеет цвет (colors[neighbor] != -1), добавляем этот цвет в множество used. Множество used хранит все цвета, которые нельзя использовать для вершины v.
 2. Назначение минимального доступного цвета: начинаем с цвета 0 и увеличиваем его, пока не найдём цвет, отсутствующий в used. Это и будет минимальный доступный цвет для текущей вершины. В коде это реализуется в цикле: while color in used.
 3. Результат: Возвращаем массив colors, где индекс — номер вершины, значение — её цвет.
 
3)Временная сложность: O(n + m), где n — количество вершин, а m — количество рёбер в графе.
  
4)Почему: Внешний цикл for v in range(n) проходит по всем вершинам, что занимает O(n) итераций. Для каждой вершины v внутренний цикл for neighbor in adj_list[v] проходит по всем её соседям. В сумме по всем вершинам это составляет O(m) операций, так как каждое ребро учитывается ровно дважды (один раз для каждой из двух вершин, которые оно соединяет). Таким образом, доминирующим фактором в сложности является количество рёбер m, и общая сложность алгоритма составляет O(n + m).
  
5)11. Имитация отжига. Объясните принцип работы метаэвристики "Имитация отжига" и роль параметра температуры.
Алгоритм имитации отжига (Simulated Annealing, SA) — метаэвристический метод оптимизации, вдохновлённый физическим процессом отжига металлов. Его цель — найти близкое к оптимальному решение в сложных задачах с большим пространством поиска, избегая застревания в локальных оптимумах.
Принцип работы «Имитации отжига»: на этапе «разогрева» — активно исследуются разные варианты решений (даже худшие), чтобы не «застрять» в локальном оптимуме; на этапе «охлаждения» — поиск фокусируется на улучшении уже найденных решений.
Ключевые шаги алгоритма:
Начало: задаём начальное решение и высокую начальную температуру T.
Цикл: генерируем новое решение — немного изменяем текущее; считаем разницу качества решений ΔE=E(новое)−E(текущее); если новое решение лучше (ΔE < 0) — принимаем его; если новое решение хуже (ΔE ≥ 0) — принимаем с вероятностью P = exp(−ΔE / T); уменьшаем температуру T по заданному графику (например, T = α · T, где 0 < α < 1).
Остановка: когда выполнено условие (максимум итераций, температура ниже порога и т. п.).
Роль параметра температуры T: температура управляет балансом между исследованием и уточнением. Высокая T: вероятность принять худшее решение близка к 1; алгоритм широко исследует пространство решений, «перепрыгивая» через локальные оптимумы; предотвращает преждевременную сходимость. Низкая T: вероятность принять худшее решение стремится к 0; алгоритм фокусируется на локальном улучшении; решение «затвердевает» (как кристаллическая решётка при охлаждении).
